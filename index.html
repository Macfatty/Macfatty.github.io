CI/CD Pipeline

Annos is built with a CI-first workflow where every code change is treated as something that should be checked, validated and made easier to deploy safely.

Git push → GitHub Actions → linting → tests → TypeScript validation → Docker build → security audit → build summary

Current CI

  • Linting and code quality checks
  • TypeScript validation
  • Backend test execution
  • Docker build verification
  • Security-oriented checks in pipeline

Next CD Steps

  • Image versioning
  • Push images to GHCR
  • Automated deployment flow
  • Health check verification
  • Rollback-oriented deployment strategy

The goal is not only fast feedback, but a delivery process that is traceable, repeatable and safer to operate over time.

Security

Security in Annos is treated as part of the system design rather than something added at the end. The project combines application- level protections with operational safeguards to reduce risk and make the platform safer to run.

Application Security

  • HTTP-only cookies instead of localStorage tokens
  • JWT authentication
  • Role-based access control
  • Rate limiting on authentication routes
  • Password requirements and validation
  • Standardized API error responses
  • GDPR data export endpoint
  • GDPR account deletion endpoint

Operational Security

  • Non-root Docker containers
  • Docker network isolation
  • .env values kept outside Git
  • No secrets exposed in frontend code
  • Health checks for service verification
  • Controlled deployment direction and rollback planning

The goal is to build a system that is easier to trust, easier to maintain and less dependent on risky shortcuts.
